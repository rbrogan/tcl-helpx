About Helpx

Warning!

Helpx is currently in a pre-release state. Up until v1.0, there may be breaking changes in each release. After v1.0, any breaking changes will result in a new major version release.

What is Helpx for?

Helpx is for generating and accessing help information in various ways (CLI, HTML pages, .chm file).

Note that to use Helpx, you must also have Gen installed, a general utility library which Helpx uses.

How does Helpx work now?

For Helpx, each package that uses it will have its own help database file. Each package will open a database connection to that database file and register that connection with Helpx. This is what the Gen package does when it is loaded. After that, when you call Helpx from the command line, it will search that database for help info and then format it for the terminal and print it out.

With this current release, you now can populate the help database mentioned above. You can then bundle helpx with your packages and give your users CLI access to help info.

How will Helpx work in the future?

In future releases, that same database will be used for things like generating HTML pages. Likewise, we anticipate adding more advanced search capabilities.

Note that it is likely that the database schema will change in future release. That will result in a major version change (same as if we had made a breaking API change). You can expect that if the schema changes then we will also provide a script to use to migrate your data, if that is at all possible.
