Helpx Database Schema

You do not have to work with the Helpx API to build a Helpx database. This document explains to you the Helpx database schema, so that you can create and modify the database in your own way, if you prefer.

Table or Column Meaning
commands This table holds all the command names and IDs. The command IDs are used by all other tables.
id Simple ID. Uniquely identifies entry.
name Fully qualified name of the command. If the command is in a namespace then it should include that namespace (e.g. MyNS::MyCommand). This is what helpx is going to search for when you run helpx SomeCommand from the CLI.
commandinfo This table holds various single pieces of information (e.g. signature, more info, etc.).
id Simple ID. Uniquely identifies entry.
commandid Refers to entry in commands table.
infotype Which type of information (e.g. signature, short description, etc.)
command_args This table holds information about the arguments to each command.
id Simple ID. Uniquely identifies entry.
commandid Refers to entry in commands table.
argname Name of the argument as it appears in the signature.
argdesc Description of the argument.
position In what order this element appears.
command_examples This table holds the examples associated with each command.
id Simple ID. Uniquely identifies entry.
commandid Refers to entry in the commands table.
data Content of the example.
position In what order this element appears.
command_seealso This table holds the other commands that are related to this command.
id Simple ID. Uniquely identifies entry.
commandid Refers to entry in the commands table.
data Content of the see also link.
position In what order this element appears.

To better understand how this works, you can look at the database file bundled with the Helpx package and compare that with the contents of the Helpx manual. The Helpx manual was generated from that data.
